# Domain Docs

This repo uses a single-context domain docs layout.

Before exploring, engineering skills should read:

- `CONTEXT.md` at the repo root, if it exists.
- `docs/adr/`, if it exists, for architectural decisions relevant to the work.

If these files do not exist, proceed silently. The domain-modeling flow can create them later when project language or decisions are resolved.

When output names a domain concept, use the vocabulary from `CONTEXT.md`. If a proposed change contradicts an ADR, surface that conflict explicitly.
